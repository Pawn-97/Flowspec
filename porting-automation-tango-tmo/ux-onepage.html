porting-automation-tango-tmo

01

我们要解决什么

admin 在现有 Port number 入口自助完成 eSIM 跨国 porting，复用 business address，全程跟踪结果。
真问题不是接一个 API，而是把分散在工单/门户里的 eSIM porting 收成一条 admin 可自助、可跟踪的流程：统一前置采集，再按国家决定自动提交还是回退工单。重心在采集统一性 + 状态可见性。
主要用户：企业 admin（兼管通信） 还会影响：porting team、持号员工、KYC team
02

线上现状

  • 现有 port-in 仅支持 Local / Toll-free 自助导入；eSIM / mobile 今天不支持。
  • 现有 Port number 向导：选类型 → 输号码 → Check portability（按 carrier 分组）→ LOA + Invoice → 提交。
  • 现有 Port history 状态模型：Submitted / Port scheduled (FOC) / Completed / Exception / Canceled。
  • UK eSIM 前置链：KYC → Get Number 取临时号 → eSIM Management 建 subscriber(user+package+临时号) → 提交；提交后 porting team 手动 Number Swap + Tango 提交。
  • business address + KYC 是 porting 前置，KYC 异步验证（Pending→Verified）。

04

明确不做

  • S7 cutover 兜底 — 依赖 Tango 成功回调（future state，拿不到实时完成态），v1 做不可靠；归运维/支持升级。
  • S11 员工感知 — 面向终端员工的沟通，不在这个 admin 项目范围。
  • S12 取消进行中 port — 次要 + Tango API 是否支持 cancel 未知，v1 先不押（留 Q4）。
  • 前置创建流程 — 创建 subscriber/address/KYC/取号按 D4 只记录、不设计；flow 内只给"去哪建"的提示出口。

额外信息

硬约束

  • business address + KYC 前置，异步验证（Pending→Verified）
  • temp number 须先建并同步 Tango，cutover 替换不可再用；eSIM 不变
  • 建 subscriber 须选 user+package+临时号；Zoom 唯一入口
  • PAC 30 天有效、须匹配；20+ 用 bulk PAC；8AM–8PM UK
  • Zoom 导入自动检测 carrier + 校验资格；swap+提交 API 自动化
  • eSIM QR 激活与 port 完成解耦；提交须前一工作日 4:30PM 前
