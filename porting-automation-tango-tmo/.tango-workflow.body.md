## **Purpose and Scope** 

This document defines the end-to-end operational workflow for porting Zoom Mobile UK numbers in partnership with Tango Networks. It establishes a unified process covering subscriber provisioning, temporary number assignment, and port request execution. The document also clarifies system behaviors for number management, including temporary number reuse, eSIM activation, and procedures for both port-in and port-out scenarios.

**Intended Audience:** Internal Mobility Support Team

## Process Overview

When a customer purchases the Zoom Phone Mobile UK SKU and requests number porting, the port-in request cannot occur immediately. The UK PAC process requires each subscriber to be provisioned in Tango with a temporary number before submitting the port request.

To meet this requirement, the customer first creates a verified Business Address and subscribers in the Zoom Web Portal via eSIM Management using temporary numbers from the Zoom number pool.

At port cutover, the temporary number is removed from the Tango Control Panel and cannot be used again once it has been replaced by the ported-in number.

### **Handling Rejections**

**If the port request shows a validation failure in the Status column:**

If a port request is rejected:

- Check the rejection reason (e.g., incorrect PAC, expired PAC, number mismatch).

<!-- -->

- Correct the issue and **resubmit immediately**—no waiting period required.

<!-- -->

- Tango does not send direct notifications; Zoom must self-serve by checking the portal [\[2\]](https://docs.zoom.us/doc/8nmBIeFcTOeJD_TWcSZWiA).

<!-- -->

- There is no restriction on the number of port-in orders that can be submitted in a single day.

### **Bulk PAC Codes (Multiple Numbers with One PAC Code)**

- The customer's carrier may issue a **single PAC code** for multiple numbers (as a general rule, mobile network operators (MNOs) will issue a single bulk PAC code when porting 20 or more numbers).

<!-- -->

- When a bulk PAC code is issued, include the same PAC code for the batch upload, Tango will accept that same PAC code for all numbers in the submission.

<!-- -->

- Ports can be phased (e.g., 25 numbers one week, 25 the next)

<!-- -->

- The PAC code remains valid for its entire validity period (~~typically~~ 30 days), giving you flexibility to phase the porting process according to your needs.
