# Require downstream implementation evidence

Flowspec v0.1 handoff will require downstream prototype agents to report implementation evidence for each FlowSpec component: the component id, implemented shadcn block or component, any deviation from the requested `shadcn_target`, and `custom_reason` when a custom component is used. `prototype-handoff.md` will also include an acceptance checklist so shadcn usage can be reviewed without turning Flowspec itself into a prototype QA tool.
