# Add path scope, prototype surface, and artifact contract

Flowspec v0.1 will add `path_scope`, `prototype_surface`, `artifact_contract`, and `acceptance_contract` to every `ux-flow-spec.json`. These fields make happy-path coverage, shadcn prototype constraints, skeleton status, handoff-vs-audit output rules, and downstream implementation evidence requirements validateable from the source-of-truth JSON instead of relying on Markdown-only conventions. `prototype_surface` defines global prototype strategy, while concrete shadcn/custom targets live on individual components rather than in a top-level component mapping. `artifact_contract` travels with each FlowSpec so a downstream agent receiving only `flow-spec/handoff/` can still know that the handoff folder is the complete default reading surface and that internal audit files are non-default evidence.
